用 Hexo + gitub 搭建个人博客

1. 准备

（此教程仅限于windows）

安装 Node.js 和 Git 。

Node.js（直接点击进入官网下载）

Git （直接点击进入官网下载）

安装的时候都安装在D盘或者其他相同的盘，安装node.js时记得勾选Automatically install the necessary tools.其他的一直点击next就行了。

安装完成后，按WIn+R输入cmd并打开，输入node -v 回车，如果显示版本号则安装成功，同上输入npm -v，git --version 看是否显示版本号。

2.连接Github

使用邮箱注册 GitHub 账户（用户名不要随便创建，尽量带_或-等特殊符号，方便后面创建仓库），选择免费账户（Free），并完成邮件验证，不会可以去b站搜索，Github因为是国外的，不爬梯子（vpn）会很慢，只有等等，最好是去接或者买个vpn，后面都会用到的，不然会很难受。（Github账号密码记着）

在刚刚安装了Git和Node.js的磁盘（刚刚说的 D盘）新建一个文件夹随便命名如 Hexo-Blog（用来存放 Hexo 的程序文件）。

打开该文件夹，右键 -> Git Bash Here。设置用户名和邮箱，分别输入下面的代码并且回车，用户名填刚刚创建Github的用户名，邮箱也是。

创建SSH密钥：

输入 ssh-keygen -t rsa -C "GitHub 邮箱"，然后一路回车。

添加密匙：

进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开id_rsa.pub 文件并复制里面的内容。

登陆 GitHub ，主页右上角头像进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

Title 随便取个名字，粘贴刚刚复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

验证连接：

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。最后显示 “Hi xxx! You've successfully……” 即连接成功。

弄完了Git Bash不用关

3.创建 Github Pages 仓库

GitHub 主页右上角加号 -> New repository：

  • 点击绿色的new创建仓库

  • Repository name 中输入 用户名.github.io（github.io一定要输入）（用户名一定要和你最上面创建Github账户的用户名一样，如果发现被重名了只有重新更改之前的Github用户名直到不会重名，更改方法去查）

  • Description 选填

  • 选public

  • 选Add a REDEME file

填好后点击 Create repository 创建。

4.本地安装 Hexo 博客程序

点开刚刚创建的文件夹如 Hexo-Blog，右键 -> Git Bash Here（如果刚刚没有关闭就不需要这一步）

有vpn的话输入npm install -g hexo-cli安装Hexo。

如果没有vpn的话输入这个npm install -g hexo-cli --registry https://registry.npm.taobao.org 会安装的快些，这个是用了淘宝镜像。

安装完后依次输入下面的指令并且回车

hexo s之后会显示http://localhost:4000这串，浏览器访问这个地址，出现 Hexo 默认页面，本地博客安装成功！但是还没有搭好，现在只是个静态界面

回到Git Ctrl+C 关闭服务器

5.部署 Hexo 到 GitHub Pages

本地博客测试成功后，就是上传到 GitHub 进行部署，使其能够在网络上访问。

首先在Git上面安装 hexo-deployer-git：

在你的Hexo-Blog文件夹中找到**_config.yml** 文件，右键用记事本打开，翻到最下面文件末尾的 Deployment 部分，修改成如下：

①要换成你的github仓库地址，打开你刚刚创建Github仓库，点击绿色的code，复制https那一串网址，填入①中

②要看你刚刚绿色code左边的branch左边那个是什么，如果是main就输入main，是其他的就输入其他的

完成后在Git上输入 hexo d回车 将网站上传部署到 GitHub Pages。

如果报错了出现error：spawn failed...

  1. 在Hexo-Blog文件夹中删除.deploy_git文件夹;
  2. 在Git中输入git config --global core.autocrlf false
  3. 然后，依次执行：
    hexo clean
    hexo g
    hexo d

基本上就完成了，

完成！这时访问我们的 GitHub 域名 https://用户名.github.io 就可以看到 Hexo 网站了。（正常情况下打开后可能就是个仓库名称，因为主题有问题，所以要去安装主题）
